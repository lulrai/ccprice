Bot Commands

All the commands in the bot is listed here with their description:


Default Prefix:      -!

Index:


[ ]      =   Optional

< >    =   Required


  Setting Commands

Configuring the settings for this guild/server requires Bot Admin or Admin or Administrator role OR Administrator permission for the user to be recognized as an administrator.

Required Bot Permissions:
  1. READ MESSAGES
  2. SEND MESSAGES
  3. EMBED MESSAGES

Commands:
  • setprefix <new_prefix>

    • Lets you set a new prefix for your server!
    Eg: -!setprefix --

  • command enable <cmd>

    • Enable a command that has been disabled on the server. Use commands disabled to check the disabled list.
    Eg: -!command enable wyr

  • command disable <cmd>

    • Disable a command that has been enabled on the server. Use commands enabled to check the enabled list.
    Eg: -!command disable cat


      Administrator Commands

    Requires Bot Admin or Admin or Administrator role OR Administrator permission for the user to be recognized as an administrator.

    Required Bot Permissions:
    1. READ MESSAGES
    2. SEND MESSAGES
    3. EMBED MESSAGES
    4. KICK (optional)
    5. BAN (optional)
    6. MANAGE ROLES
    7. MANAGE NICKNAMES

    Commands:
  • cstmcmd add <cmd_name> <cmd_text>

    • Lets you add a custom command with the provided name and value.
    Eg: -!cstmcmd add dirt I love dirt!

  • cstmcmd remove <cmd_name>

    • Removes the custom command with the provided name.
    Eg: -!cstmcmd remove moo

  • cstmcmd set <cmd_name> <new_cmd_text>

    • Lets you reset the value of the provided command without having to delete it and readd it again.
    Eg: -!cstmcmd set dirt I love dirt even more!

  • kick <@user>

    • Kicks the mentioned user. Note that the bot doesn't have ban/kick permission by default.
    Eg: -!kick @Raizusekku

  • ban <@user>

    • Bans the mentioned user. Note that the bot doesn't have ban/kick permission by default.
    Eg: -!ban @Raizusekku

  • guildinfo

    • Provides all the necessary and detailed information about the guild you are in.
    Eg: -!guildinfo

  • joinrole [@user]

    • Set a join role or remove it. Note that it might be case sensitive.
    No arguments to remove it.
    With arguments (role) to set that as the role.
    Eg: -!joinrole Member

  • asr <role_name>

    • Adds the provided role (name) to the self-assignable roles list.
    Eg: -!asr Dirt

  • rsr <role_name>

    • Removes the provided role (name) from the self-assignable roles list.
    Eg: -!rsr Dirt

  • medals add <emoji> <multiplier> <medal_name>

    • Adds a medal (emoji) with a multiplies and a name.
    Emoji      = Actual emoji such as :pizza: (Note that custom emojis DO NOT work).
    Multiplier = Number denoting how many medals instead of having the same medal numerous times; such as :pizza:x2 or :pizza:x2.
    Name        = Name of the medal that you want to add.
    Eg: -!medals add :pizza: 5 PizzaMaster

  • medals remove <medal_name>

    • Removes the medal with the provided medal name.
    Eg: -!medals remove PizzaMaster


      Moderator Commands

    Requires Bot Mod or Mod or Moderator role for the user to be recognized as a moderator.

    Required Bot Permissions:
    1. READ MESSAGES
    2. SEND MESSAGES
    3. EMBED MESSAGES
    4. MANAGE NICKNAMES
    5. MANAGE CHANNELS
    6. MANAGE MESSAGES
    7. ADD REACTIONS
    8. MOVE MEMBERS
    9. MUTE MEMBERS
    10. DEAFEN MEMBERS

    Commands:
  • mute <@user>

    • Mutes the mentioned user in the current channel only!
    Eg: -!mute @Raizusekku

  • unmute <@user>

    • Unmutes the mentioned user from the channel.
    Eg: -!unmute @Raizusekku

  • award <@user> <medal>

    • Awards the mentioned user with the provided medal name. Use medallist command to view the medals and the medal names.
    Eg: -!award @Raizusekku PizzaMaster

  • clear [@user] [num]

    • Clears the given amount of messages in the channel with or without arguments (Bot can only access the past 2 weeks of messages so it cannot clear anything older than that)
    Valid sub-commands:
    clear               = Clears past 100 messages.
    clear <num>         = Clears the given amount of messages.
    clear <@user>       = Clears past 100 messages of the mentioned user.
    clear <@user> <num> = Clears the given amount of messages of the mentioned user.
    Eg: -!clear @Raizusekku 50

  • move <@user> <chan_id>

    • Moves the given user from the channel they are in to the provided channel id. (Make sure the user is in the voice chat.)
    Eg: -!move @Raizusekku 304821843192315904

  • move all <from_chan_id> <to_chan_id>

    • Moves all the users from the provided channel id to the given channel id. (Because why not?)
    Eg: -!move all 304821843192315904 240614697848537090

  • infect <@user> <emoji>

    • Makes the bot add the provided emoji to the tagged user until they are disinfected. Note that custom emojis do work for this command unlike medal commands.
    Eg: -!infect @Raizusekku :cow:

  • disinfect <@user>

    • Simply disinfects an infected user and saves their lives, definitely.
    Eg: -!disinfect @Raizusekku


      Informative Commands

    Useful commands for you to use for various purposes.

    Required Bot Permissions:
    1. READ MESSAGES
    2. SEND MESSAGES
    3. EMBED MESSAGES
    4. CREATE INSTANT INVITES

    Commands:
  • ping

    • Displays the ping of the bot, very simple!
    Eg: -!ping

  • usr [@user]

    • Displays the roles of a mentioned user or yourself.
    No arguments = Roles of yourself.
    W/ arguments = Roles of the mentioned user.
    Eg: -!usr @Raizusekku

  • userinfo [@user]

    • Displays info about a mentioned user or yourself.
    No arguments = Info about yourself.
    W/ arguments = Info about the mentioned user.
    Eg: -!userinfo @Raizusekku

  • inrole <@user>

    • Shows a list of users who are in the provided role name.
    Eg: -!inrole @DirtIsAwesome

  • botstats

    • Displays all the info required about the bot.
    Eg: -!botstats

  • invite [server]

    • Provides you with a list of invite links present in the server or the invite link for the bot.
    No arguments = Invite link for the bot.
    W/ arguments = Invite links for the current server (The actual word 'server').
    Eg: -!invite server

  • weather <city_name>

    • Provides you with the weather info about the given city.
    Eg: -!weather LA

  • time <city/address>

    • Provides you with the current time based on the given info.
    Eg: -!time LA

  • feedback <stuffs>

    • You can provide feedback about the bot to the owner of the bot! Just don't make it empty lol. You may indicate whatever you didn't like about the bot.
    Eg: -!feedback I didn't like something.

  • help

    • Displays the list of commands and info about it in the client itself.
    Eg: -!help

  • commands <all/enabled/disabled>

    • Shows the list of commands that are either enabled, disabled or all of them.
    enabled  = List of enabled commands.
    disabled = List of disabled commands.
    all        = List of all the commands. Eg: -!commands all

  • suggest <stuffs>

    • Similar to feedback but it's only for suggestion of improvements in the bot or this website!
    Eg: -!suggest It needs more Dirt.

  • medallist

    • Displays a list of all the medals available on the server.
    Eg: -!medallist

  • selfroles

    • Displays the list of all the self assignable roles set on the server.
    Eg: -!selfroles

  • customcommands

    • Displays the list of all the custom commands present on the server.
    Eg: -!customcommands


      Fun Commands

    Just some fun/useful commands because why not, right?

    Required Bot Permissions:
    1. READ MESSAGES
    2. SEND MESSAGES
    3. EMBED MESSAGES
    4. MANAGE ROLES

    Commands:
  • wyr

    • Just a fun little game of Would You Rather. Feel free to suggest more games.
    Eg: -!wyr

  • feed <@user> <foodname>

    • Feed the mentioned user with the provided food name. Most/All of the food emojis should work.
    Eg: -!feed @Raizusekku pizza

  • poke <@user>

    • Poke someone! :D
    Eg: -!poke @Tori

  • giveself <role_name>

    • Adds a self-assignable role to your roles. Check the list of self-assignable roles using selfroles command.
    Eg: -!giveself DirtMaster

  • removeself <role_name>

    • Removes a self-assignable role from your roles. Check the list of self-assignable roles using selfroles command.
    Eg: -!removeself NotDirtMaster

  • cat

    • A random image of one or many cute cats!
    Eg: -!cat

  • dog

    • Random and cute image of dogs! <3
    Eg: -!dog

  • say <stuffs>

    • Make the bot repeat what you say. If you have MANAGE MESSAGE permission then it deletes your message so that it looks even more legit!
    Eg: -!say Beep beep ima sheep

  • poll [title] <option1|option2>

    • Creates a poll based on the given arguments.
    title is an option argument and is not required.
    Separate the title and the options with the delimiter "|" for each option you put in.
    Eg: -!poll Title is Dirt | Dirt is awesome | Dirt is cool | Oh wait.. Dirt is always amazing
